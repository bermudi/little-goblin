## Phase 11: Dreaming pipeline (adapted, not ported — see decision 0027)

- [ ] Create `src/memory/dreaming.ts` — `DreamingPipeline` class with `runLightSleep()`, `runRemSleep()`, `runDeepSleep()` methods. This is an adaptation of openclaw's dreaming phase structure, NOT a port — the promotion mechanism uses model-opinion confidence instead of openclaw's 2932-line recall store. Each method: build phase-specific prompt, call `sessionManager.ensureInternal("__goblin_dreaming__")` to get the session state, call `dispatcher.enqueueInternalTurn(state, prompt, onComplete, onError)` to dispatch. Light sleep: read transcript after cursor (`memory-dreaming-cursor.json`), filtered to the lookback window (default 24 hours; effective range is `max(cursor, now − lookback)` to `now`). In `onComplete(text)`: parse JSON candidates (`text`, `category`, `confidence`, `target`, `rationale`) by extracting the first fenced JSON code block; quarantine malformed output. Filter `category = "skip"` or candidates below the confidence threshold and quarantine with reason `low_confidence`. Dedupe via cosine similarity (threshold 0.85); near-duplicates update the existing entry (preserve `created_at`/`source_session`, refresh `updated_at`/`updated_source_session`) rather than inserting a new row. Promote novel candidates with `origin = "dreaming"` and `promoted_at` set. On first observation of a session: if `memory-reflection.json` exists, migrate its cursor value to `memory-dreaming-cursor.json` (same line offset) and remove the old file; if no cursor file exists, seed cursor to transcript end (no backfill). REM sleep: aggregate concept tags across distinct sessions, detect recurring themes, promote to the scope with the most frequent origin session (ties by most recent `updated_at`, then scope name ascending; default `general`). Deep sleep: promote short-term entries to durable, update `promoted_at`, run budget compaction, update dream diary. In `onError(err)`: log the error, advance cursor if applicable, do not promote.
