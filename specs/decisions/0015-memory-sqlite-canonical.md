# Memory SQLite canonical

## Status

accepted

## Context

The existing memory system stores curated facts in markdown files (`memory.md` capped at 4000 chars, `user.md` capped at 2000 chars) with `\n§\n` delimiters. It performs lexical substring search, regex-based reflection, and injects the full memory store as a per-turn aside. It also creates a git commit after every successful `memory_write`.

These mechanisms break down as memory grows:
- The 6000-char hard cap cannot be relaxed without per-file bookkeeping.
- Lexical search cannot match semantically related but token-disjoint queries.
- Per-turn full snapshots are a fixed and growing token tax.
- Git-per-write adds I/O and complexity for a single-user, single-process agent.

A SQLite-backed engine with hybrid search, transcript indexing, and scheduled consolidation addresses these issues.

## Decision

- `little-goblin` SHALL store canonical memory in a SQLite database at `$GOBLIN_HOME/state/memory/memory.sqlite`.
- Markdown files (`user.md`, `general/memory.md`, `topics/<chatId>/<topicId>/memory.md`, `agents/<name>/memory.md`) SHALL be export-only artifacts regenerated by the `memory export` CLI. Direct edits to markdown SHALL NOT be reflected in the store.
- Per-file character caps (4000/2000) SHALL be replaced by a single global character budget (default 50,000) across curated memory entries.
- Per-write git commits in `$GOBLIN_HOME/state/memory/.git` SHALL be removed. History SHALL be captured by `created_at`/`updated_at` columns and git history at the repository level.
- Entry delimiters (`\n§\n`) SHALL be replaced by SQLite rows.

## Consequences

- `memory_write` persists immediately to SQLite; `memory export` is required to view markdown.
- Manual edits to markdown will be overwritten on the next export.
- The global budget requires compaction logic that drops oldest dreaming-promoted entries first while preserving user-authored entries.
- `AGENTS.md` and canon specs must be updated to remove delimiter/cap/git-commit requirements.
- The schema and export mapping must support `user`, `general`, `topics/*`, `agents/*`, `archive/*`, and `transcript/*` scopes.
